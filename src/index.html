Understanding Git Tags

A simple explanation of tags in Git.

What is a Git Tag?

A Git tag is a reference that points to a specific commit. Tags are commonly used to mark release versions such as v1.0.0.

Types of Git Tags

1. Lightweight Tag

A lightweight tag is just a pointer to a commit. It does not store extra metadata.

2. Annotated Tag

An annotated tag stores additional information such as the tagger name, email, date, and a message.
